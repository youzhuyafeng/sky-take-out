JavaWeb苍穹外卖练习项目（仅后端）

2026.5.6开始，2026.5.21结项，计15天

作为练手项目，除了java基础语法和库之外，主要用于熟悉一些第三方的javaAPI和Mybatis作为半自动的ORM框架的在操作时一些细节问题，“纸上得来终觉浅”，听课跟上手写代码是两回事，至少对于我来说大部分Debug的时间基本都用于与Mybatis斗智斗勇了（笑）。

该项目大部分代码属于基础的CRUD功能，少量涉及WebSocket、Spring Task、AOP、Redis技术，整体难度不大。于我而言其实该项目的最大收获其实是终于对git命令体系比较熟悉了，之前基本只会git add . git commit git push git pull 几个简单的命令，写这个项目时遇到的一些问题让我不得不深入了解一下git怎么用......这个项目有个配置文件里面会存一些密钥啥的，直接push到远程仓库会被阿里云拦截，然后黑马提供的初始代码框架中的.gitignore文件中又没有把这个配置文件排除，当时我也不懂这些，我一通add commit下来push的时候直接被拦截了，我还傻乎乎的只是把密钥删了又提交一遍，还是不行；问AI才知道git提交时会提交完整的修改路径，阿里云会检查所有的提交，只删除密钥不过是掩耳盗铃，必须使用reset修改头指针回到这个配置文件还不包含阿里云密钥的提交状态，然后commit的时候把这个密钥文件修改提交排除，不提交修改到本地版本库，pull的时候就不会出事了（暂时的，因为每次提交都得这么做），再然后为了偷懒寻找解决办法——之前我在gitignore中排除了这个配置文件但是没生效，这是因为gitignore不会对已跟踪的文件生效，所以需要先取消git对这个文件的跟踪，然后git add的时候就不会包含这个被排除的配置文件了，就能一劳永逸了。除此之外黑马提供的部分涉及数据库操作的代码不是那么高效，我需要额外开分支去优化代码，这就又用到git checkout git merge这些命令了......

碎碎念就到这了。总之作为第一个项目还是让我收获颇丰😊
